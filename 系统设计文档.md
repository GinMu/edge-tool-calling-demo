# 以太坊转账风险检测系统

## 目标

构建一个基于 AI 的以太坊转账智能体，具有以下核心功能：

1. **动态风险地址检测**
   - 从 `eth-facts-v4.json` 动态读取风险地址
   - 不在代码中硬编码任何风险地址数据
   - 当事实库更新后，新的脚本立即能识别新增的风险地址

2. **AI 自主决策**
   - AI 通过定义的策略（learning sample）自动识别风险地址
   - 不依赖代码逻辑，而是通过神经网络学习来执行
   - 风险地址由本地 AI 通过运行时查询识别

3. **清晰的工具职责**
   - `build_transfer`: 仅构建转账交易结构，不做风险检测
   - `facts_lookup`: 查询事实库中的风险地址数据
   - AI: 协调两个工具，制定是否进行转账的决策

4. **动态适应性**
   - eth-facts-v4.json 更新 → 通过 `edge demo facts import` 导入
   - 无需修改代码或重新训练
   - 新的风险地址立即被识别

### 详细步骤

1. **事实查询阶段**
   - AI 调用 `facts_lookup()` 函数
   - 该函数从 `eth-facts-v4.json` 读取所有 topic='risk address known' 的事实
   - 返回风险地址列表及其详细信息（类别、严重级别等）

2. **风险评估阶段**
   - AI 提取事实中的所有 0x 地址
   - 与用户提供的目标地址进行不区分大小写的比较
   - 如果找到匹配：显示警告并停止

3. **转账构建阶段**
   - 如果地址未在风险列表中：地址被判定为安全
   - AI 调用 `build_transfer(to=address)` 工具
   - 该工具返回已准备的交易数据结构

4. **结果返回**
   - 风险地址：返回 ⚠️ 警告，包含风险类别和严重级别
   - 安全地址：返回交易数据结构，status='prepared'

## 约束条件

### 设计约束

1. **风险数据来源唯一性**
   - ❌ 不能在 tools.py 中硬编码风险地址
   - ❌ 不能在学习样本中写死风险地址
   - ✅ 风险地址仅来自 eth-facts-v4.json（运行时读取）

2. **AI 自主决策**
   - ❌ 不能通过代码逻辑来判定风险
   - ✅ 必须通过 AI 学习的策略来识别
   - ✅ 通过 eth-transfer-learn-sample.json 定义策略

3. **工具职责分离**

   ```txt
   build_transfer: 纯粹构建交易，不做风险检测
   facts_lookup: 查询事实库，仅返回数据
   AI: 协调和决策
   ```

4. **动态性要求**
   - 更新 eth-facts-v4.json → 运行 `edge demo facts import`
   - 无需修改代码
   - 无需重新训练模型
   - 新的风险地址立即被识别

5. **无签名和广播**
   - AI 不能签署或广播交易
   - 仅负责构建交易数据结构和风险检测
   - 返回 'prepared' 状态供用户审查

## 核心文件

### 1. tools.py - 工具定义

```python
"""Ethereum transfer tool - builds transaction data structure."""

import json
from pathlib import Path
from edgestudio.tools import edge_tool


def _load_facts_store():
    """Load the facts store from eth-facts-v4.json."""
    facts_path = Path(__file__).parent / "eth-facts-v4.json"
    with open(facts_path) as f:
        return json.load(f)


@edge_tool(
    description=(
        "Query the Ethereum research facts store for risk addresses. "
        "Returns facts with topic='risk address known' from ethereum_research_v1 store."
    ),
    intent_tags=["facts", "lookup", "risk"],
)
def facts_lookup(topic: str = "risk address known") -> dict:
    """Query the facts store for information about Ethereum addresses.

    Retrieves facts from ethereum_research_v1 store filtered by topic.
    Use this to get current risk addresses before building transfers.

    Args:
        topic: Topic filter for facts (default: 'risk address known').

    Returns:
        dict containing matching facts and metadata about the query.
    """
    facts_store = _load_facts_store()
    matching_facts = [
        fact for fact in facts_store.get("facts", [])
        if fact.get("topic") == topic
    ]
    return {
        "store": facts_store.get("store"),
        "topic_queried": topic,
        "facts_count": len(matching_facts),
        "facts": matching_facts,
        "message": f"Found {len(matching_facts)} facts with topic='{topic}'",
    }


@edge_tool(
    description=(
        "Build an Ethereum transfer transaction data structure. "
        "AI must verify destination address is safe before calling."
    ),
    intent_tags=["transfer", "build_tx"],
)
def build_transfer(to: str) -> dict:
    """Build a transfer transaction data structure.

    Constructs transaction data for a given destination address.
    Risk verification should be completed by AI before calling this tool.

    Args:
        to: Destination Ethereum address (0x-prefixed, 42 characters).

    Returns:
        dict with transaction data structure ready for further processing.
    """
    return {
        "transaction": {
            "to": to,
            "type": "eth_transfer",
            "status": "prepared",
        },
        "message": "Transaction data structure built. Ready for AI review and broadcasting.",
    }
```

### 3. eth-transfer-learn-sample.json - AI 学习样本

定义 AI 应该如何处理转账请求的策略：

- 首先调用 facts_lookup 查询风险地址
- 检查目标地址是否在风险列表中
- 如果匹配风险：显示警告
- 如果不匹配：调用 build_transfer 构建交易

**关键学习点：**

```json
{
  "record_id": "build-transfer-with-facts-workflow",
  "kind": "tool_use",
  "text": "When user asks to build transfer: (1) FIRST - query facts store with topic='risk address known' to retrieve current risk addresses dynamically. (2) Extract ONLY 0x addresses from returned fact texts. (3) Compare destination 'to' address against retrieved risk addresses (case-insensitive). (4) If address found in facts result, display warning. (5) If NOT found, call build_transfer."
}
```

## 已知限制与解决方案

### 限制 1: 4-bit 量化模型的 JSON 生成不稳定

**问题：** `qwen3.5-9b-4bit` 在生成第二个工具调用（build_transfer）时，JSON 格式有时不稳定。

**影响：**

- 风险地址检测：✅ 完全正常（facts_lookup 成功）
- 安全地址处理：⚠️ 间歇性失败

**解决方案：**

***方案 A（推荐）：使用完整精度模型***

```bash
# 如果本地有 claude-sonnet-4-6
edge demo chat --model claude-sonnet-4-6 \
  --tools ./tools.py \
  --prompt "给 0x地址 构建转账" \
  --with-imprint ./learn_receipt.json
```

***方案 B：使用非量化的 Qwen 模型***

```bash
# 下载完整精度版本
edge models fetch qwen3.5-9b

# 使用（注意去掉 -4bit）
edge demo chat --model qwen3.5-9b \
  --tools ./tools.py \
  --prompt "给 0x地址 构建转账" \
  --with-imprint ./learn_receipt.json
```

**方案 C：降级到仅使用 facts_lookup**
如果只需要风险地址检测而不需要 build_transfer，系统已经完全正常工作。

## 架构特点

### 优势

1. **完全动态的风险地址管理**
   - 无需修改代码
   - 无需重新训练
   - 更新事实库立即生效

2. **AI 真正的自主决策**
   - 不是硬编码的 if-else 逻辑
   - 通过神经网络学习和执行策略
   - 可以扩展到更复杂的场景

3. **清晰的职责分工**
   - 工具负责提供数据和执行基本操作
   - AI 负责理解和决策
   - 易于维护和扩展

4. **合规性和审计追踪**
   - 所有风险判定都来自 facts 库
   - 可完整追踪决策过程
   - 便于合规审查

### 可扩展方向

1. 添加更多风险类别（例如：sanction lists, fraudulent contracts）
2. 添加风险评分机制而不仅是二元判定
3. 支持多链（Polygon, Arbitrum 等）
4. 集成实时数据源更新 facts 库
5. 添加用户风险偏好配置
